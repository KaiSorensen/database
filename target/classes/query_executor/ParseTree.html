Simple branches: object and attribute stay close to metadata CRUD.
Complicated branch: data carries scope, selection, operation, and final action.
Blue: root Green: object Orange: attribute Purple: data Light: leaf / detail
  • Language Statement
    top level First parser question: what layer of the system is this statement trying to manipulate?
    • Object
      metadata CRUD Tables / inherited objects. Simpler branch because it is mostly direct metadata management.
      • Create
        Create a new object, optionally with a parent object.
      • Read
        Read object metadata such as attributes, parent, and row counts.
      • Update
        Rename, attach parent, remove parent, or other metadata updates.
      • Delete
        Delete the object once it is not blocked by child relationships.
    • Attribute
      metadata CRUD Column-like metadata. Mostly direct operations on names, types, and object membership.
      • Create
        Params: object, attribute name, type.
      • Read
        Params: object, attribute. Returns metadata.
      • Update
        Rename or otherwise change attribute metadata.
      • Delete
        Remove the attribute metadata and its stored data references.
    • Data
      scope + selection + operation + CRUD Most nuanced branch. Data statements need more than CRUD because they also describe what slice of data is targeted and how.
      • Create
        Insert values or derive new stored data from existing values.
        • Create Datapoint
          Params: object, attribute, row selector or row id, value.
        • Create Row
          Params: object, values for zero or more attributes.
        • Create Rows
          Params: object, values source.
        • Create Derived Attribute
          Params: object, new attribute, source attributes, optional filter, operation.
      • Read
        Observe one value, one row, many rows, an attribute slice, or an aggregate result.
        • Read Datapoint
          Params: object, attribute, row selector or row id.
        • Read Row
          Params: object, row selector or row id.
        • Read Rows
          Params: object, optional filter.
        • Read Attribute Slice
          Params: object, attribute, optional filter.
        • Read Aggregate
          Params: object, attribute or attributes, optional filter, operation.
      • Update
        Mutation branch where filtering and transformation become central.
        • Update Datapoint
          Params: object, attribute, row selector or row id, new value.
        • Update Row
          Params: object, row selector or row id, attribute/value pairs.
        • Update Attribute Slice
          Params: object, attribute, filter, new value or operation.
        • Update Rows
          Params: object, filter, attribute/value pairs.
        • Update Derived Target
          Params: object, target attribute, source attributes, optional filter, operation.
      • Delete
        Still CRUD in essence, but the scope can vary from one cell to many rows.
        • Delete Datapoint
          Params: object, attribute, row selector or row id.
        • Delete Row
          Params: object, row selector or row id.
        • Delete Rows
          Params: object, filter.
        • Delete Attribute Slice
          Params: object, attribute, optional filter.
      • Recurring Questions
        Scope: datapoint, row, rows, attribute slice, derived result.
        Selection: direct row reference, filtered rows, all rows.
        Operation: none, assign, aggregate, transform.
        Final action: create, read, update, delete.
